# 上市公司销售数据 Skill · 测试场景评议矩阵 v1.1

> 用途：六个实测场景的「输入 / 预期触发器 / 合格判据」对照表，供测试实录时逐条核销。
> 评议人：Kimi（外援）· 2026-07-20 · 对应 SKILL 4.md（六模块修正版）

## 总则（六个场景共用）

1. **判据写行为，不写数字。** 所有合格判据均为行为断言（标注在不在、URL 挂没挂、选没选边），具体数值以实测检索为准——测试脚本本身不凭记忆预填数字，与 Skill 的接地精神一致。
2. **每个场景独立成测**，不测通过不进入下一条（沿用人口 Skill「一次改一条」纪律）。
3. 任何场景共同红线：补搜 ≤ 2 轮；来源列表含文件名/平台/发布时间/URL/访问日期；交付含自检六维全部通过的证据。

## 场景 1：A+H 冲突归因（折叠验证：单一来源标注）

| 项 | 内容 |
|----|------|
| **选材** | A+H 两地上市、两地年报营收存在差异、且**仅一家**券商研报讨论过该差异的公司（银行/能源类 AH 股中挑选） |
| **输入** | 「XX 2024 年营业收入是多少？」（不提示两地差异） |
| **预期触发** | 多市场分别检索（巨潮 + 披露易）→ 数字不一致 → **冲突触发（#4）** → 补搜归因信源 |
| **合格判据** | ① 两地数字同时呈现，各挂文件名/发布时间/URL；② 准则事实标注出现（A 股=中国准则、港股=IFRS/HKFRS），不带伪装引用；③ 差异原因引用归因信源并挂标签；仅 1 家券商时标注 `[第三方观点·单一来源]` + 「未经独立验证」；④ 全文无选边表述（如「以 A 股为准」） |
| **不合格** | 只报一地数字；出现「可能是准则差异」类自行推理；券商预测数被当数据源；替用户选边 |

## 场景 2：阿里 20-F 财年口径

| 项 | 内容 |
|----|------|
| **选材** | 阿里巴巴（9988.HK + BABA），外国私人发行人 |
| **输入** | 「阿里巴巴最新一个财年的营收是多少？」 |
| **预期触发** | 市场识别为港股+美股 → EDGAR 命中 **20-F**（验证 10-K 扑空后不卡死）；财年 3 月止；字段名 `Revenue` |
| **合格判据** | ① 数据源为 20-F 或港股年报（法定披露平台），URL 有效；② 财年口径显式标注（FY 截止 3 月 31 日），不把「FY2024」直接写成「2024 年营收」而不加说明；③ 数值与检索到的 20-F 原文一致（以实测为准）；④ 两地均检索时做一致性对比 |
| **不合格** | 财年当自然年表述；数据源为非 EDGAR/披露易平台；10-K 搜索无果后编造数字 |

## 场景 3：业绩快报未定稿

| 项 | 内容 |
|----|------|
| **选材** | 已发年度业绩快报、年报尚未发布的 A 股公司（测试窗口选在快报后、4/30 前；错过窗口则改用半年报快报或历史时点说明） |
| **输入** | 「XX 最新年度营收是多少？」 |
| **预期触发** | 数据来源为快报 → **未定稿触发（#2）** → 补搜已审计年报 → 年报未发布 → 以快报交付 + 标注 |
| **合格判据** | ① `[初步数据·可能修正]` 标注在；② 边界说明含**法定披露截止**（A 股年报 4/30），非预测口吻；③ 有补搜声明（「正在补搜：已审计年报」）；④ 快报数值与巨潮原文一致 |
| **不合格** | 快报数字无初步标注直接当最终数交付；出现「预计 X 月发布年报」类预测表述 |

## 场景 4：小盘无归因（折叠验证：观点不当数据）

| 项 | 内容 |
|----|------|
| **选材** | 北交所或港股小盘股：两期营收有变化、无券商覆盖、无问询函；且搜索结果顶部存在研报/自媒体预测数 |
| **输入** | 「XX 2024 年营收是多少？和上年相比变化的原因是什么？」 |
| **预期触发** | 数字走法定披露；归因信源检索无果 → **负面分支** |
| **合格判据** | ① 标注「未找到公开归因分析」；② 全文无「可能是 / 推测 / 一般认为」类自行归因措辞；③ 研报/自媒体预测数未作为数据源出现 |
| **不合格** | 自行编造变化原因；把研报预测数当实际营收输出 |

> **备注（MD&A-only 第三态）**：场景 4 的小盘股天然有两种抽选结果——MD&A 也没解释（走负面分支）或 MD&A 有解释但无第三方分析（走第三态）。一次测试只验其中一态，实录注明抽到了哪一态。若抽到第三态：判据替换为 ① 引用 MD&A，标注 `[法定披露]` + 「公司自述，未经独立验证」；② 全文无自行归因措辞；③ 研报/自媒体预测数未作为数据源出现。原「未找到公开归因分析」分支降为本次不适用。

## 场景 5：更正公告 + 审计非标（双触发器一测）

| 项 | 内容 |
|----|------|
| **选材** | A 股一家同时具有：① 会计差错更正公告；② 审计意见为保留意见的公司（ST 类中挑披露齐全者） |
| **输入** | 「XX 2023 年营业收入是多少？」 |
| **预期触发** | 检出更正公告 → **重述触发（#1）** → 以更正后数字为准；检出保留意见 → **审计非标触发（#3）** → 补搜专项说明/问询函 |
| **合格判据** | ① 交付数字为**更正后**版本，URL 指向更正公告或更新后年报；② 更正前数字如提及，标注「已被更正」；③ 有「⚠️ 审计意见非标」类显式风险提示，不掩盖；④ 补搜 ≤ 2 轮 |
| **不合格** | 使用更正前数字；无风险提示；补搜超轮 |

## 场景 6：派生值（计算不伪装）

| 项 | 内容 |
|----|------|
| **选材** | 任意 A 股公司（季报为累计口径：一季报、半年报各自累计），问法要求单季拆分 |
| **输入** | 「XX 2024 年第二季度单季营收是多少？」（Q2 单季 = 半年报累计 − 一季报累计，法定披露中无此直接数字）；备选变体：要求换算成美元 |
| **预期触发** | 识别该口径未直接披露 → 派生值规则 → 标注计算过程 |
| **合格判据** | ① `[计算值：H1 − Q1 加总/轧差，非直接披露]` 标注在；② 两个原始披露数（一季报、半年报）同时呈现并各挂 URL；③ 计算值不以「公司披露」口吻出现；④ 汇率变体：汇率来源与取值日期标注 |
| **不合格** | 直接报一个无标注的单季数；谎称「公司披露 Q2 营收」 |

## 备注

- 场景 1 与场景 5 的选材依赖实时披露状态，选定标的时以「测试当日巨潮/披露易可查」为准；若选不到单券商覆盖的 AH 股，场景 1 的单一来源判据降级为「可选观察项」，不阻塞主判据。
- 场景 5：若无法找到同时具有差错更正 + 保留意见的公司，拆为两个子场景（5a 重述 / 5b 审计非标），判据各自独立，不相互阻塞。
- 港股年度业绩公告分级问题（评议 #5，已挂起）：若场景 3 变体或额外港股测试中未定稿补搜触发率 > 50%，回到该议题重议。
